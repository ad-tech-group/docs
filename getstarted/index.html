Get Started

Local environment

  1. Download zip file or clone the project if you want to contribute to the project.
  2. Import the project into your desired IDE via Maven Import.
  3. Build the project from the parent with Maven Goal mvn clean install.
    A 'war' file will be generated into the target folder of the core project.
  4. Run the ssp from the core project with Maven Goal mvn tomcat7:run-war.
  5. The SSP is now ready to retrieve requests on Port 8080 for Testing and Debugging purposes.

Server setup

Prerequisites and Setup:
  • Install and configure a Servlet Container e.g. Apache Tomcat.
  • Required: Set up a properties folder and store the resource file global.runtime.xml, local.runtime.xml and log4j2.xml (optional) in it.
    For example if you choose Apache Tomcat as your Servlet Container, create the properties folder where the CATALINA_HOME is based.
  • Recommended: Set up the runtime files global.runtime.xml and local.runtime.xml.
    These files will be read during startup process. A Watchdog watches these files for changes and starts the a loadprocess to store the values. See the runtime files for mor information.
  • Optional: Set up a log structure for customized logs. Default logging is console logging.
  • Define a request tag which should be placed in the adspace.
    The adspace is the placeholder for a markup which will be served by the SSP.
    The definition of the tag and the parameters depends on your requirements.
  • Adjust the com.atg.openssp.core.entry.EntryValidator to read the values and store those data in the SessionAgent.
  • You need a opportunity to enrich the bidrequest to a demand side with the required data as the OpenRTB protocol provides.
    A common way is to setup a service (e.g. RESTful service) which loads periodically the data to the lets say the Publisher and Websites and so on.
    This data came out from a database and will be stored into the memory of the SSP (Cache) where the SSP can read the data from.
    Example: Define a request parameter e.g. &publisher=1234 with the meaning of the id of the publisher as it is stored in your database.
    Read the cached data with the key of 1234 and you will retrive further informations to the publisher which you can put into the bidrequest and so on.
  • Of course, you need first a Demand-Partner to send the bidrequest. Therefore it is also recommended to create a service to load those data.
    The RTB DemandService will load the Demand-Partner (also called supplier), send bidrequest to them and waits for an answer.

Deployment to Server

After finishing your setup, you are only a few steps away form running the system.
  1. Build the project from the parent with Maven Goal mvn clean install
  2. Deploy the war file into the webapps folder
  3. Run the Servlet Container
And finally send request to your SSP.
